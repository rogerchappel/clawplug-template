# Security Policy

## Supported Versions

The `main` branch receives security fixes for the template. Generated plugins should define their own support policy before publication.

## Reporting a Vulnerability

Please report suspected vulnerabilities through GitHub Security Advisories or a private maintainer report with reproduction steps, affected files, and expected impact.

Do not include real API keys or production credentials in reports. Use placeholder values in template fixtures and examples.

## Security Model

This repository is a starter template for OpenClaw plugins. It should not ship real credentials, production endpoints, or live side effects in the example plugin.
